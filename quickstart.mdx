CC Design activates automatically when you send a design-related request to Claude Code — there is no slash command to remember and no mode to switch into. Describe what you want to build, mention a brand if you have one in mind, and CC Design handles the rest: asking clarifying questions, loading the right references, building the HTML, verifying it with Playwright, and delivering the file.

## Your first prompt

Start with a straightforward request. CC Design works with natural language — describe the output type and the context:

```
"Design a landing page for our SaaS product"
"Create a 10-slide pitch deck for the Q3 board meeting"
"Build an interactive prototype of the checkout flow"
"Explore 3 visual directions for the new dashboard"
```

CC Design will ask a few clarifying questions before building — about output format, fidelity level, number of variations, and any existing design systems in your project. Answer what you know; leave the rest and it will make reasonable choices.

## Brand style cloning

To apply a specific brand's visual system, mention the brand name in your prompt. CC Design detects brand names and loads the corresponding design system from [getdesign.md](https://getdesign.md) before writing any code:

```
"Create a pricing page with Stripe's aesthetic"
"Notion-style kanban board"
"Mix Vercel's minimalism with Linear's purple accents"
"Show me this dashboard in Apple style vs Tesla style"
```

Over 68 brands are supported. The skill extracts colors, typography, and component patterns from the brand's design system and applies them throughout the artifact.

## How CC Design handles your request

<Steps>
  <Step title="Understand">
    CC Design asks clarifying questions about output format, fidelity level, number of variations, constraints, and existing design systems. If a brand name appears in your prompt, it's flagged as a brand style clone task immediately.
  </Step>

  <Step title="Route">
    The skill reads its routing table and identifies which reference documents and templates apply to your task type. Only the relevant files are loaded — for example, a slide deck request loads `references/starter-components.md` and copies `templates/deck_stage.js` to your project directory. A brand clone additionally loads `references/getdesign-loader.md`.
  </Step>

  <Step title="Acquire context">
    CC Design searches your workspace for existing design system files (`DESIGN.md`, token files, HTML/CSS). If your project already has a visual vocabulary, it reads and reuses it. If not, it works from the brand system or asks you for a starting point.
  </Step>

  <Step title="Design intent">
    Before writing any code, CC Design answers a 6-question checklist from `references/design-excellence.md`: focal point, emotional tone, visual flow, spacing strategy, color strategy, and typography hierarchy. This step shapes every decision in the build.
  </Step>

  <Step title="Build">
    The HTML artifact is written. React components are embedded inline using Babel when needed (with pinned library versions). Multiple design directions are exposed as in-page tweaks rather than separate files, so you can explore variations without managing multiple outputs.
  </Step>

  <Step title="Verify">
    Three-phase verification runs via Playwright MCP:
    - **Structural** — no console errors, layout intact, responsive behavior correct
    - **Visual** — screenshot taken and reviewed for rendering issues
    - **Design excellence** — hierarchy clarity, spacing consistency, color harmony, and emotional fit checked against the design intent from step 4

    If any phase fails, CC Design fixes the issue and re-verifies before continuing.
  </Step>

  <Step title="Deliver">
    The verified file is handed off with a brief summary of caveats and suggested next steps. The filename is descriptive (e.g., `Pricing Page.html`, not `untitled.html`).
  </Step>
</Steps>

## Tips for best results

<AccordionGroup>
  <Accordion title="Be specific about brand elements">
    Instead of "make it look like Stripe", try "create a pricing page with Stripe's aesthetic — clean white background, their signature purple gradient on the CTA, and their table layout for plan comparison". The more detail you give, the less CC Design needs to guess.
  </Accordion>

  <Accordion title="Combine brand and task type">
    Brand cloning and output format work together. "Build a 6-slide pitch deck in Notion's style" is a valid single prompt — CC Design handles the routing for both the deck format and the brand system simultaneously.
  </Accordion>

  <Accordion title="Request comparisons explicitly">
    CC Design generates 3+ design directions by default, but you can ask for specific comparisons: "Show me this dashboard in Apple style vs Tesla style" or "Explore 3 visual directions — one safe, one bold, one unconventional". Directions are exposed as tweaks in a single file.
  </Accordion>

  <Accordion title="Let it ask questions">
    The clarifying questions at the start of each task are part of the workflow, not a limitation. Answering them accurately leads to better output. If you're in a hurry, you can say "use your best judgment" and CC Design will proceed with reasonable defaults.
  </Accordion>
</AccordionGroup>
