## How it works

<Steps>
  <Step title="Install the skill">
    Clone `cc-design` into `~/.claude/skills/` and run `npm install` in the `scripts/` directory to enable export features.
  </Step>
  <Step title="Send a design prompt">
    Describe what you want to design — mention a brand name for style cloning, or describe the output format and content.
  </Step>
  <Step title="Skill routes and builds">
    CC Design classifies your task, loads only the relevant references and templates, acquires context from your project, and builds the HTML artifact.
  </Step>
  <Step title="Verify and receive">
    Three-phase Playwright verification checks for console errors, visual correctness, and design excellence before the file is delivered.
  </Step>
</Steps>

## Key capabilities

<AccordionGroup>
  <Accordion title="Progressive context loading">
    The main `SKILL.md` stays under 200 lines. Twelve technical reference documents are loaded on demand based on the routing table, keeping context window usage minimal across all task types.
  </Accordion>
  <Accordion title="Brand style cloning">
    Mention any of 68+ brand names (Stripe, Vercel, Notion, Apple, Tesla, Linear…) and the skill fetches that brand's `DESIGN.md` from getdesign.md, extracts colors, typography, and component patterns, and applies them to your artifact.
  </Accordion>
  <Accordion title="Tweaks system">
    When you need multiple design directions, a self-contained in-page controls panel is embedded directly in the HTML. Variants are stored in `localStorage` so exploration happens in one file, not many.
  </Accordion>
  <Accordion title="Three-phase verification">
    Every artifact is verified structurally (no console errors, responsive layout), visually (screenshot review), and for design excellence (hierarchy, spacing, color harmony, emotional fit) before delivery.
  </Accordion>
  <Accordion title="Export to PPTX, PDF, and inline HTML">
    Node.js scripts convert verified HTML to editable PowerPoint, printable PDF via Playwright, or a fully self-contained single-file HTML with all assets inlined.
  </Accordion>
</AccordionGroup>
